# linux-learning-journey
Day 1- Today I started my journey learning the Linux command line. I practiced the basics that let me navigate and manage files from the terminal: pwd, ls, mkdir, cd, and touch. Small commands, big confidence — I’m building foundational skills for automation, scripting, and system work. I’ll be documenting progress here as I learn new commands.
Day 2 — Linux Fundamentals. Focused on file management and inspection using cp, mv, rm, and cat to copy, relocate/rename, remove, and view file contents from the shell. Consolidating core workflow skills built on Day 1 and preparing for permissions, searching with grep, and basic shell scripting. Documenting progress and practical examples. Feedback welcome.
Day 3 — Linux Fundamentals. Today I covered identity, session control, privilege escalation, and permissions with whoami, clear, sudo, and chmod. I practiced confirming the active user (whoami), clearing the terminal (clear), running commands with elevated privileges (sudo), and modifying file permissions (chmod). I also learned that chmod 777 grants read, write, and execute to owner, group, and others — effectively making the file accessible and modifiable by anyone, so use it cautiously for security reasons. Next: deeper permission concepts (umask, setuid/setgid), ACLs, and safe sudo practices.
